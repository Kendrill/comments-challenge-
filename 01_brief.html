Аналитический блэкаут

Задание для команды · Восстановить картину закупочного процесса компании из сырых логов SAP

🚨 Ситуация
Кризис

48 часов назад компания «НоваТех» подверглась масштабному каскадному сбою. Цепная реакция отказов, начавшись с серверов аналитики, уничтожила все дашборды, отчётность и аналитические системы. Единственное, что уцелело — сырые логи из ERP-системы SAP по модулю закупок (MM).

Руководство компании не понимает, как реально работает процесс закупок: где задержки, где нарушения, где теряются деньги. Ваша задача — провести процесс-майнинг по сырым данным и восстановить полную картину.

Ваша задача

С помощью AI-чата проанализировать event log из SAP, восстановить карту процесса Procure-to-Pay, найти узкие места и нарушения, рассчитать ключевые метрики и оформить результаты в виде HTML-отчёта.

Что нужно знать до старта
Что такое Process Mining?
Анализ реальных бизнес-процессов по логам из IT-систем. Вместо того чтобы спрашивать людей «как устроен процесс», мы смотрим на данные и видим, как процесс работает на самом деле.
3 вида Process Mining
1. Discovery — восстановить карту процесса из логов
2. Enhancement — найти bottleneck-и и метрики
3. Conformance — сравнить «как есть» vs «как должно быть»
Правильная последовательность
1. Загрузить данные → понять структуру
2. Discovery → построить карту
3. Enhancement → метрики и bottleneck-и
4. Рекомендации → выводы для бизнеса
5. Визуализация → HTML-отчёт

⏱️
Длительность шагов
Считайте разницу между timestamps последовательных шагов. Где задержки? Для каких заводов/поставщиков?
🔄
Циклы и rework
Повторяющиеся активности в одном кейсе (PR Rejected → PR Created снова) — это rework. Сколько процентов кейсов?
⚠️
Пропущенные шаги
Есть ли кейсы, где пропущены обязательные шаги? PO без PR? GR до PO?
👻
Zombie-кейсы
Кейсы, которые начались, но никогда не дошли до Payment Completed. Где они застряли?
🤖 Примеры промптов для AI
Что спросить у AI-чата

Копируйте и адаптируйте. Чем точнее промпт — тем лучше результат.

🔍 Шаг 1 — Разведка данных ▼
Я загружаю SAP P2P event log (CSV). Проанализируй его: 1. Сколько уникальных кейсов (Case_ID)? 2. Какие активности (Activity) встречаются и как часто? 3. Какой временной период охватывают данные? 4. Какие заводы (Plant), поставщики (Vendor), закупочные группы (Purchasing_Group) есть? 5. Какой диапазон сумм (Net_Value)? Дай краткую сводку в виде таблицы.
🗺️ Шаг 2 — Карта процесса ▼
Построй карту процесса (process map) по этим данным: 1. Определи основной happy path (наиболее частая последовательность шагов) 2. Покажи все ветвления и альтернативные пути 3. Отметь циклы (rework loops) — когда процесс возвращается на предыдущий шаг 4. Укажи частоту каждого перехода (сколько кейсов идут по каждому пути) Визуализируй как граф (можно текстовый или Mermaid-диаграмму).
🔍 Шаг 3 — Bottleneck-и и аномалии ▼
Проведи глубокий анализ bottleneck-ов и нарушений: 1. BOTTLENECK-И: Для каждого перехода между шагами рассчитай медианное и среднее время. Где самые большие задержки? Разбей по Plant и Vendor. 2. НАРУШЕНИЯ: - Maverick Buying: есть ли кейсы с PO Created, но без PR Created? - Retroactive PO: есть ли случаи, где Goods Receipt раньше PO Created? - Duplicate Invoices: есть ли кейсы с несколькими Invoice Receipt? - Zombie Cases: какие кейсы не дошли до Payment Completed? 3. ПАТТЕРНЫ ПО ИЗМЕРЕНИЯМ: - Какой Plant самый проблемный? - Какой Vendor самый проблемный? - Какая Purchasing Group самая медленная? Приведи конкретные цифры и Case_ID примеров.
📊 Шаг 4 — Метрики и рекомендации ▼
Рассчитай ключевые Process Mining метрики: 1. Lead Time (end-to-end): среднее, медиана, мин, макс (от первого до последнего события в кейсе) 2. Lead Time по шагам: средняя длительность каждого перехода 3. Throughput: сколько кейсов завершено за месяц 4. Rework Rate: % кейсов с повторными активностями 5. Maverick Buying Rate: % кейсов без PR 6. Process Completion Rate: % кейсов, дошедших до Payment На основе этих метрик дай 5 конкретных рекомендаций руководству НоваТех. Каждая рекомендация должна содержать: что исправить, кому адресована, ожидаемый эффект.
🎨 Шаг 5 — HTML-отчёт ▼
Создай красивый HTML-отчёт для руководства НоваТех. Один HTML-файл со встроенным CSS и JS. Секции отчёта: 1. Заголовок: «Process Mining Report — NovaTech Procurement (P2P)» 2. Executive Summary: 3-4 ключевых вывода 3. Process Map: визуальная схема процесса (используй Mermaid.js через CDN) 4. Bottleneck Analysis: таблица с задержками по шагам 5. Violations Found: список нарушений с количеством и примерами 6. Key Metrics: карточки с основными показателями 7. Recommendations: пронумерованные рекомендации Стиль: тёмная тема, современный дизайн, аккуратная типографика. Подключи Mermaid.js через CDN: https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.min.js

📖 Ключевые термины ▼
Event LogЖурнал событий — основа для process mining. Каждая строка = одно событие (что, когда, кто)
Case / Case IDОдин экземпляр процесса от начала до конца (одна заявка на закупку)
ActivityШаг процесса (событие): «Purchase Order Created», «Goods Receipt»
Happy PathИдеальный путь — последовательность шагов без отклонений
BottleneckУзкое место — шаг, на котором процесс тормозит (большое время ожидания)
Lead TimeВремя от начала до конца кейса (end-to-end)
ReworkПовторение шага — процесс «откатывается назад» (заявка отклонена → переделана)
ConformanceСоответствие реального процесса эталонному регламенту
VariantУникальная последовательность шагов. 100 кейсов могут иметь 15 разных вариантов

⚙️ Как работать с AI: лучшие практики ▼

1. Загрузите файл, а не копируйте текст — AI лучше работает с прикреплённым CSV, чем с вставленным текстом.

2. Идите по шагам — не просите всё сразу. Сначала разведка, потом анализ, потом визуализация.

3. Просите конкретные цифры — «Покажи топ-3 bottleneck-а с медианным временем в днях» лучше, чем «найди проблемы».

4. Для HTML-отчёта — попросите AI сделать «один HTML-файл со встроенным CSS и JS». Это легче скопировать и открыть.

5. Используйте артефакты — если AI поддерживает артефакты (Claude, ChatGPT Canvas), попросите рендерить HTML прямо в чате.

⭐ Бонусное задание
Conformance Checking (задание со звёздочкой)

Запросите у фасилитатора файл reference_process.md — внутренний регламент закупок НоваТех. Загрузите его в AI-чат вместе с event log и проведите conformance checking:

  • Сравните фактический процесс с каждым правилом регламента
  • Посчитайте % нарушений по каждому правилу (SLA, обязательный тендер, запрет retroactive PO...)
  • Визуализируйте: какие правила нарушаются чаще всего
  • Дайте рекомендации по приведению процесса в соответствие
Промпт для AI: Я загружаю два файла: 1. sap_p2p_event_log.csv — реальные данные процесса 2. reference_process.md — внутренний регламент Проведи Conformance Checking: - Проверь каждое правило из регламента - Посчитай % кейсов, нарушающих каждое правило - Покажи конкретные примеры нарушений (Case_ID) - Визуализируй результат в виде таблицы: Правило | Target | Actual | Статус - Добавь результат в HTML-отчёт как отдельную секцию
