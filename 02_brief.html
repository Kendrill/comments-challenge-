Раздатка для участников

Станция: «Коммуникационный коллапс»

Сюжет. У компании «НоваТех» отключены внутренние каналы связи. Сотрудники не получают обновлений, менеджеры теряют управляемость, руководство требует оперативной информации.

Вы — антикризисный штаб. Ваша задача — за ограниченное время подготовить проект Telegram-бота кризисной коммуникации.

Ваша задача

  • отправлять экстренные оповещения;
  • собирать обратную связь;
  • проводить быстрые опросы;
  • поддерживать коммуникацию по ролям;
  • при возможности — делать сводки для руководства.

Важно

  • Код вручную писать не нужно
  • Вся работа идёт через ML-чат
  • На итоговой проверке через агента проект будут анализировать по файлам
  • И запускать, чтобы проверить, как он работает в реальности

Бонус: +1 балл

Можно получить +1 балл сверх основной оценки, если реализовать LLM-чатбота, который:

  • общается человеческим языком, а не только командами;
  • помогает в кризисной коммуникации;
  • дополняет основной сценарий, а не заменяет его.

Примеры LLM-функций

  • ответы на вопросы сотрудников в свободной форме;
  • суммаризация сообщений;
  • генерация краткой сводки для руководства;
  • классификация проблем;
  • помощь в подготовке кризисных апдейтов.

Рекомендуемая стратегия

Сначала сделайте базовый MVP.

  • команды;
  • роли;
  • понятную структуру;
  • README;
  • нормальные тексты.

Чек-лист перед сдачей

Базовый уровень

  • Это Telegram-бот
  • Есть /start
  • Есть /help
  • Есть /status
  • Есть /alert
  • Есть /poll
  • Есть /feedback
  • Есть роли пользователей
  • Есть requirements.txt
  • Есть .env.example
  • Есть README.md
  • Тексты на русском и подходят под кризисную ситуацию

Усиление

  • Есть хотя бы 1 дополнительная функция
  • или есть LLM-чатбот / LLM-функция

Главный совет

Не пытайтесь сделать всё.

Лучше собрать хороший базовый бот, чем сделать много функций, которые не выглядят рабочими.
