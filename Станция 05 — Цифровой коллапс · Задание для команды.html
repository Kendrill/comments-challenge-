🚨 Ситуация
Кризис

48 часов назад компания «НоваТех» подверглась масштабному каскадному сбою. Цепная реакция отказов, начавшись с серверов аналитики, распространилась на все ключевые системы компании. Корпоративный сайт полностью недоступен. Клиенты и партнёры не могут получить информацию о компании. Нужно показать миру: компания работает.

Ваша задача

С помощью AI-инструментов создать красивый, адаптивный сайт для компании «НоваТех» и задеплоить его — сделать доступным по публичной ссылке. Сайт должен выглядеть профессионально, внушать доверие и быть доступен всему миру.

📝 Алгоритм работы
Рекомендуемые шаги (20 минут)
  1. Обсудите концепцию (2 мин) — Какой стиль? Тёмная/светлая тема? Одна страница или несколько? Распределите роли: кто пишет промпт, кто деплоит, кто готовит демо.
  2. Составьте промпт и сгенерируйте сайт (8 мин) — Используйте формулу из шпаргалки ниже. Чем точнее опишете — тем лучше результат. Попросите AI создать HTML + CSS + JS.
  3. Доработайте и протестируйте (5 мин) — Проверьте мобильную версию (сузьте окно браузера). Добавьте недостающие секции. Попросите AI исправить то, что не нравится.
  4. Задеплойте сайт (3 мин) — Опубликуйте сайт на любом бесплатном хостинге, чтобы он стал доступен по публичной ссылке. Скопируйте ссылку.
  5. Подготовьтесь к демо (2 мин) — Откройте сайт по ссылке, проверьте на телефоне, подготовьте рассказ.
💡 Лайфхак: используйте Agent Mode (Cmd+I / Ctrl+I) в IDE для генерации сайта. Можно попросить AI сразу сделать несколько страниц с общим CSS и JS.

💡 Подсказка про функционал приёма заявок

Чтобы получить 4-й балл, сайту нужна форма заявки + админ-панель для их просмотра. Вот архитектура того, что нужно построить.

Архитектура решения: 3 компонента
🌐
Frontend

  • + Форма: имя, email, сообщение
  • + Отправка данных по API
  • + Сообщение «Заявка отправлена!»
→
⚙️
Backend (API)
Сервер, который принимает и отдаёт данные
  • + POST /api/requests — сохранить заявку
  • + GET /api/requests — список заявок
  • + Авторизация для админки
→
🗄️
База данных
Хранилище заявок
  • + Таблица: id, имя, email, текст, дата
  • + Таблица: admin_users (логин, пароль)
🔐 Отдельная страница /admin — таблица заявок, доступна по паролю
Какой стек попросить у ИИ
Вариант 1: Самый простой

Один HTML + JSON-файл. Заявки хранятся в localStorage браузера или в JSON-файле. Админ-панель — отдельная HTML-страница с паролем.

Промпт ИИ: «Добавь форму заявки, которая сохраняет данные в localStorage. Сделай страницу /admin.html с паролем, которая показывает все заявки в таблице.»

✅Работает без сервера. Минус: данные только в одном браузере.
Вариант 2: С бэкендом

Frontend + Python/Node.js сервер + SQLite. Сервер принимает заявки, сохраняет в базу, отдаёт список в админку.

Промпт ИИ: «Создай Python Flask API с двумя эндпоинтами: POST /api/requests (сохранить заявку в SQLite), GET /api/requests (вернуть все заявки). Добавь простую авторизацию. Сделай admin.html с таблицей заявок.»

🚀Полноценное решение. Нужен запущенный сервер (Python/Node.js).
Как это выглядит для пользователя
1️⃣
Клиент открывает лендинг → заполняет форму (имя, email, «Хочу консультацию») → нажимает «Отправить»
2️⃣
Система сохраняет заявку в базу / localStorage → показывает «Спасибо, мы свяжемся!»
3️⃣
Админ открывает /admin.html → вводит пароль → видит таблицу всех заявок с датами
⚡Совет: начните с Варианта 1 (localStorage) — это занимает 5 минут через ИИ. Если останется время — попросите ИИ переделать на Вариант 2 с сервером.

📂Какие бывают сайты

Чтобы говорить на одном языке — вот основные типы сайтов и чем они отличаются.

Landing Page (лендинг)

1 страница, одна цель: продать, собрать заявки, зарегистрировать. Вся информация — сверху вниз, без сложной навигации.

Промо-акции, подписки, продукты

Корпоративный сайт

5–20 страниц: главная, о компании, услуги, кейсы, контакты, блог. Задача — доверие и информирование.

Банки, консалтинг, производство

Интернет-магазин

Каталог товаров, корзина, оплата, личный кабинет. Тысячи страниц. Ключевая метрика — конверсия (% купивших).

Ozon, Wildberries, Amazon

Web-приложение

Интерактивный продукт в браузере. Пользователь работает, а не читает: создаёт, редактирует, управляет.

Figma, Notion, Google Docs

SaaS-сайт

Маркетинговый сайт для продвижения веб-приложения: hero + features + pricing + testimonials.

stripe.com, slack.com

Портал / Dashboard

Внутренний интерфейс: аналитика, админ-панели, CRM. Фокус на информационной плотности и эффективности.

Salesforce, Jira, Grafana

Профессионально
Много «воздуха» между элементами. Страница дышит
Любительски

Профессионально
2 цвета + нейтральный фон. Один акцент для кнопок
Любительски
Радуга из 5+ цветов, кислотные оттенки
Профессионально
Макс. 2 шрифта. Чёткая иерархия размеров
Любительски
3+ шрифтов, нет системы размеров
Профессионально
Одна CTA-кнопка на экране. Понятно, что нажать
Любительски
5 кнопок — непонятно, что главное
Профессионально
Скруглённые углы, мягкие тени, плавные переходы
Любительски
Острые углы, резкие тени, мигающие элементы
Профессионально
Выравнивание по сетке. Всё ровно, симметрично
Любительски

🎯Главное правило: лучше меньше, но качественно. Whitespace — это не «пусто», а приём премиального дизайна. Apple, Stripe, Tesla — мастера «воздуха».
🔥Что сейчас модно

Эти приёмы можно попросить у ИИ — он их знает и умеет делать.

Glassmorphism

Полупрозрачные панели с эффектом матового стекла. Создаёт глубину.

Промпт: «glassmorphism cards with backdrop blur»

Gradient Mesh

Сложные многоцветные градиенты — «жидкие» переливы на фоне. Выглядит дорого.

Промпт: «mesh gradient background»

Bento Grid

Карточки разного размера в асимметричной сетке. Тренд от Apple.

Промпт: «bento grid layout for features»

Dark Mode

Тёмный фон, светлый текст, неоновые акценты. Tech-стильно и премиально.

Промпт: «dark theme with neon accents»

Scroll Animations

Элементы плавно появляются при прокрутке: fade in, slide up. Сайт «оживает».

Промпт: «smooth scroll animations with CSS/JS»

Микро-анимации

Кнопки реагируют на наведение, карточки увеличиваются при hover.

Промпт: «hover effects on cards and buttons»

🎨Быстрый гайд по цвету и стилю
Готовые цветовые схемы для промпта
Tech — тёмный + фиолетовый
Корпоративный — белый + синий
Финтех — чёрный + зелёный
Энергичный — светлый + оранжевый
Bold — тёмный + красный/розовый
Правила, которые нельзя нарушать
  • Максимум 2 шрифта — заголовки + текст
  • Один акцентный цвет для кнопок, остальное нейтральное
  • Минимум 16px для текста — иначе не читается на телефоне
  • Контраст — светлый на тёмном ИЛИ тёмный на светлом
  • Одинаковые отступы — 60px везде, а не 20 тут, 80 там
  • Одна primary-кнопка на экран
